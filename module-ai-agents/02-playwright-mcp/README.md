# Playwright MCP: браузерная автоматизация для агента

Playwright MCP нужен тогда, когда агент должен не просто рассуждать о веб-странице, а открыть ее, пройти по ссылкам, найти элементы и извлечь данные в структурированном виде.

## Зачем это нужно

В Threat Intelligence и SOC часто встречаются задачи, где данные находятся не в API, а в обычной HTML-странице:

- бюллетени производителя;
- страницы обновлений;
- ленты advisories;
- таблицы CVE;
- списки затронутых продуктов.

Playwright MCP дает агенту способ работать с такими источниками через браузерную автоматизацию.

Официальный источник:

- [Playwright MCP getting started](https://playwright.dev/docs/getting-started-mcp)

## Что вы изучите

- как поставить Playwright MCP;
- как подключить его к OpenCode;
- как запустить его напрямую;
- зачем нужен Chromium;
- чем Chromium отличается от Chrome;
- как диагностировать типовые ошибки.

## Практические команды

Проверить доступность Node.js:

```bash
node -v
npm -v
```

Проверить запуск сервера:

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Проверить конфиг OpenCode:

```bash
cat ~/.config/opencode/opencode.json
```
