# Установка Playwright MCP

Playwright MCP — отдельный сервер для браузерной автоматизации. Его можно подключить через конфиг OpenCode или запустить напрямую для диагностики.

Ссылки:

- [Playwright MCP getting started](https://playwright.dev/docs/getting-started-mcp)

## Почему нужен Chromium

В примерах используется `chromium`.

Зачем это нужно:

- Chromium — базовая open-source ветка, на которой построены многие браузеры;
- он широко поддерживается в автоматизации;
- его используют как стандартную среду для воспроизводимых тестов;
- он предсказуем для автоматизации.

### Чем Chrome отличается от Chromium

Кратко:

- Chromium — open-source проект;
- Google Chrome — дистрибутив Chromium с дополнительными фирменными компонентами и интеграциями;
- для автоматизации обычно достаточно Chromium;
- если вам нужно поведение именно Google Chrome, это отдельный сценарий.

Справка по Chromium:

- [Chromium project](https://www.chromium.org/Home/)

## Вариант 1. Установка через OpenCode MCP

Этот путь нужен для обычной работы: OpenCode сам запускает Playwright MCP по конфигу.

Пример конфига:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "playwright": {
      "type": "local",
      "command": [
        "npx",
        "-y",
        "@playwright/mcp@latest",
        "--browser=chromium",
        "--headless"
      ],
      "enabled": true
    }
  }
}
```

Что делает этот конфиг:

- создает локальный MCP-сервер Playwright;
- запускает его через `npx`;
- выбирает Chromium;
- включает headless-режим;
- делает сервер доступным для OpenCode.

## Вариант 2. Прямой запуск

Если вы хотите проверить сервер отдельно от OpenCode, запустите его напрямую:

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Ожидаемый результат: MCP-сервер стартует без ошибок. Если команда падает, проблема в Node.js, npm, сети или браузерных зависимостях.

Этот сценарий полезен для диагностики:

- вы отделяете проблему сервера от проблемы клиента;
- проверяете, что пакет устанавливается;
- понимаете, где именно падает запуск.

## Когда использовать headless

Headless-режим полезен, если:

- вы запускаете сервер на CI;
- не хотите открывать окно браузера;
- проверяете только извлечение данных;
- работаете в среде без GUI.

Если вам нужно визуально посмотреть на шаги агента, headless можно отключить.

## Проверка установки

Проверьте, что Playwright MCP запускается напрямую:

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Ожидаемый результат: сервер стартует без ошибок.

Проверьте, что OpenCode установлен:

```bash
opencode --help
```

Ожидаемый результат: OpenCode выводит справку.

Проверьте, что конфиг доступен:

```bash
cat ~/.config/opencode/opencode.json
```

Ожидаемый результат: в конфиге есть секция `mcp` с сервером `playwright`.
