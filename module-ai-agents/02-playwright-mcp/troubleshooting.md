# Playwright MCP: типовые проблемы и диагностика

Этот раздел нужен, потому что в практических занятиях проблемы почти всегда возникают не в самой идее, а в окружении: браузер не установлен, npm не работает, JSON сломан или сервер запущен, но клиент его не видит.

Официальные источники:

- [Playwright MCP getting started](https://playwright.dev/docs/getting-started-mcp)
- [Playwright docs](https://playwright.dev/)

## 1. `browser not installed`

### Что это значит

Сервер запускается, но нужный браузерный движок отсутствует.

### Что проверить

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Если браузерные компоненты не подтянуты автоматически, установите их согласно официальной Playwright-документации.

### Что делать

- проверьте, что установлен Node.js;
- проверьте, что `npx` работает;
- перечитайте официальный guide;
- убедитесь, что вы запускаете тот браузер, который указан в конфиге.

## 2. `playwright install`

### Что это значит

Playwright ожидает, что нужные браузерные бинарники будут установлены.

### Диагностика

```bash
npx playwright --help
```

### Что делать

Пользуйтесь официальными командами Playwright для установки браузеров. Не смешивайте ручную и автоматическую установку в одном учебном сценарии, если это не требуется.

## 3. `npm` issues

### Типовые признаки

- `npm: command not found`;
- ошибки прав доступа;
- проблемы с proxy;
- ошибки сети при `npx`.

### Диагностика

```bash
node -v
npm -v
npx -y @playwright/mcp@latest --browser=chromium --headless
```

### Что делать

- убедиться, что Node.js установлен;
- проверить PATH;
- не запускать установку через sudo без необходимости;
- перепроверить сетевой доступ к npm registry.

## 4. JSON syntax errors

### Типовые ошибки

- пропущенная запятая;
- лишняя запятая;
- строки в одинарных кавычках;
- комментарии в JSON-файле;
- неверные скобки.

### Диагностика

```bash
node -e 'JSON.parse(require("fs").readFileSync(process.argv[1], "utf8")); console.log("ok")' ~/.config/opencode/opencode.json
```

### Что делать

- используйте валидный JSON;
- если нужен JSONC, убедитесь, что конкретный клиент это поддерживает;
- тестируйте конфиг после каждого изменения.

## 5. MCP server not visible

### Что это значит

Сервер запущен, но клиент его не подхватывает.

### Что проверить

- правильное имя сервера в конфиге;
- `enabled: true`;
- правильный путь к конфигу;
- корректный `command`;
- отсутствие ошибок при старте сервера;
- перезапуск клиента после изменения конфига.

### Диагностика

```bash
cat ~/.config/opencode/opencode.json
opencode --help
```

## Практические команды

```bash
node -v
npm -v
npx -y @playwright/mcp@latest --browser=chromium --headless
cat ~/.config/opencode/opencode.json
```

## Что должно получиться

- вы умеете распознавать базовые ошибки запуска;
- вы знаете, что сначала проверять: Node.js, npm, JSON, конфиг, сервер;
- вы не путаете проблему браузера с проблемой клиента;
- у вас есть диагностический чеклист для учебной группы.

## Проверка результата

- повторно запустите сервер после устранения ошибки;
- проверьте синтаксис JSON;
- убедитесь, что клиент видит сервер;
- при необходимости сверьте параметры с официальной документацией Playwright MCP.
