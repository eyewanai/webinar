# Playwright MCP: типовые проблемы и диагностика

Раздел помогает быстро отделить проблему окружения от ошибки в задаче: браузер не установлен, npm не работает, JSON сломан или OpenCode не видит сервер.

Ссылки:

- [Playwright MCP getting started](https://playwright.dev/docs/getting-started-mcp)
- [Playwright docs](https://playwright.dev/)

## 1. `browser not installed`

### Суть

Сервер запускается, но нужный браузерный движок отсутствует.

### Что проверить

Запустите сервер напрямую, чтобы проверить браузерные зависимости:

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Ожидаемый результат: сервер стартует или показывает конкретную ошибку установки браузера.

Если браузерные компоненты не подтянулись автоматически, установите их по инструкции Playwright.

### Что делать

- проверьте, что установлен Node.js;
- проверьте, что `npx` работает;
- перечитайте официальный guide;
- убедитесь, что вы запускаете тот браузер, который указан в конфиге.

## 2. `playwright install`

### Суть

Playwright ожидает, что нужные браузерные бинарники будут установлены.

### Диагностика

Проверьте, что команда Playwright доступна через `npx`:

```bash
npx playwright --help
```

Ожидаемый результат: выводится справка Playwright.

### Что делать

Пользуйтесь официальными командами Playwright для установки браузеров. Не смешивайте ручную и автоматическую установку в одном сценарии без необходимости.

## 3. `npm` issues

### Типовые признаки

- `npm: command not found`;
- ошибки прав доступа;
- проблемы с proxy;
- ошибки сети при `npx`.

### Диагностика

Проверьте Node.js, npm и прямой запуск Playwright MCP:

```bash
node -v
npm -v
npx -y @playwright/mcp@latest --browser=chromium --headless
```

Ожидаемый результат: версии Node.js и npm выводятся, сервер стартует или показывает сетевую ошибку.

### Что делать

- убедиться, что Node.js установлен;
- проверить PATH;
- не запускать установку через sudo без необходимости;
- перепроверить сетевой доступ к npm registry.

## 4. JSON syntax errors

### Типовые ошибки

- пропущенная запятая;
- лишняя запятая;
- строки в одинарных кавычках;
- комментарии в JSON-файле;
- неверные скобки.

### Диагностика

Проверьте JSON-синтаксис конфига:

```bash
node -e 'JSON.parse(require("fs").readFileSync(process.argv[1], "utf8")); console.log("ok")' ~/.config/opencode/opencode.json
```

Ожидаемый результат: команда выводит `ok`.

### Что делать

- используйте валидный JSON;
- если нужен JSONC, убедитесь, что конкретный клиент это поддерживает;
- тестируйте конфиг после каждого изменения.

## 5. MCP server not visible

### Суть

Сервер запущен, но клиент его не подхватывает.

### Что проверить

- правильное имя сервера в конфиге;
- `enabled: true`;
- правильный путь к конфигу;
- корректный `command`;
- отсутствие ошибок при старте сервера;
- перезапуск клиента после изменения конфига.

### Диагностика

Проверьте конфиг и сам запуск OpenCode:

```bash
cat ~/.config/opencode/opencode.json
opencode --help
```

Ожидаемый результат: конфиг содержит `playwright`, а OpenCode выводит справку.

## Критерий завершения

Диагностика завершена, когда понятно, где ошибка: Node.js/npm, браузерные зависимости, JSON-конфиг или перезапуск OpenCode.
