# Типовые ошибки

Раздел помогает быстро определить, где проблема: в конфиге, Python-окружении, API-ключе, API или данных статьи.

## MCP server не виден

Что проверить:

- валидность JSON;
- абсолютный путь к Python;
- абсолютный путь к `server.py`;
- установлен ли `fastmcp`;
- активируется ли venv.

Если сервер не виден в OpenCode, почти всегда проблема в конфиге или в том, как запускается локальный процесс.

## API key не задан

Ошибка:

```text
RST_THREAT_API_KEY is not set
```

Решение:

```bash
export RST_THREAT_API_KEY="..."
```

Ожидаемый результат: переменная появляется в текущей сессии терминала.

После этого снова проверьте:

```bash
echo "$RST_THREAT_API_KEY"
```

Ожидаемый результат: терминал выводит значение ключа.

## Агент не вызывает нужный инструмент

Что делать:

- явно указать имя MCP-сервера;
- явно указать названия инструментов;
- улучшить docstring, то есть описание инструмента в коде;
- сделать описание инструмента короче и конкретнее.

Если docstring слишком общий, агент может выбрать неправильный инструмент или не понять, когда его использовать.

## Пустой JSON

Проверить:

- получил ли агент текст статьи;
- правильно ли работает регулярное выражение для IOC;
- есть ли IOC в статье;
- не потерялись ли `[.]`.

Если статья не содержит подходящих IOC, JSON может оказаться пустым не из-за ошибки, а потому что в источнике реально нет совпадений.

## Ошибка `requests`

Проверить:

- доступность API;
- ключ;
- лимиты;
- сетевые ошибки.

Если API отвечает нестабильно, сохраните raw-ответ и посмотрите, что именно вернул сервер.

## Практические команды

Проверить ключ:

```bash
echo "$RST_THREAT_API_KEY"
```

Ожидаемый результат: терминал выводит ключ или пустую строку, если он не задан.

Проверить Python-окружение:

```bash
which python
```

Ожидаемый результат: путь указывает на виртуальное окружение проекта.

Проверить JSON:

```bash
python3 -m json.tool ioc.json
```

Ожидаемый результат: команда выводит отформатированный JSON или ошибку синтаксиса.

## Проверка результата

- проблема отнесена к одной зоне: конфиг, Python-окружение, API, ключ или текст статьи;
- ключ проверен через переменную окружения;
- JSON проверен стандартной командой;
- ошибка инструмента отделена от ошибки данных.
