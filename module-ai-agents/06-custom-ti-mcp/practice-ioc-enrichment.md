# Практика: извлечение и обогащение IOC

Цель практики: собрать цепочку, где Playwright открывает статью, TI MCP извлекает и обогащает IOC, а агент сохраняет итог в JSON.

## Исходная задача

Источник:

```text
https://thedfirreport.com/2023/05/22/icedid-macro-ends-in-nokoyawa-ransomware
```

Нужно:

1. Открыть статью через Playwright MCP.
2. Получить текст страницы.
3. Извлечь IOC.
4. Обогатить каждый IOC через RST Cloud API.
5. Сохранить результат в `ioc.json`.

## Что именно считается IOC

Defanged-формат — безопасная запись IOC, где опасные элементы нейтрализованы, например `example[.]com`.

В этой задаче ищите:

- домены в defanged-формате;
- IPv4 в defanged-формате.

Enrichment, или обогащение, — это запрос к внешнему источнику за дополнительными сведениями об IOC.

Исходный defanged-вид нужно сохранить при извлечении. Нормализацию используйте только для enrichment.

## Как должен выглядеть результат

Итоговый файл должен быть JSON-массивом записей. Каждая запись должна содержать:

- исходный IOC;
- нормализованный IOC;
- тип IOC;
- данные enrichment;
- источник отчета;
- описание, если оно есть.

## Как запускать практику

1. Запустите OpenCode.
2. Убедитесь, что доступны два MCP-сервера:
   - `playwright`;
   - `ti-tools`.
3. Передайте агенту промпт из файла `prompt.md`.
4. Дождитесь создания `ioc.json`.
5. Проверьте JSON.

## Команды проверки

Проверить, что JSON валиден:

```bash
python3 -m json.tool ioc.json
```

Ожидаемый результат: команда выводит отформатированный JSON без ошибок.

Проверить количество записей:

```bash
jq 'length' ioc.json
```

Ожидаемый результат: терминал выводит число записей в массиве.

Посмотреть первую запись:

```bash
jq '.[0]' ioc.json
```

Ожидаемый результат: терминал выводит первый IOC-объект с исходным и нормализованным значением.

## Что здесь нужно понять

Раньше такой сценарий часто писали как один Python-скрипт. Здесь та же логика разделена на инструменты:

- браузерный инструмент открывает источник;
- TI MCP ищет и обогащает IOC;
- агент сам связывает шаги между собой.

Такой подход отличается от одноразового скрипта: агент сам выбирает порядок вызова готовых инструментов.

## Проверка результата

- у вас есть `ioc.json`;
- JSON проходит проверку;
- в данных нет потерянных defanged IOC;
- enrichment отработал хотя бы для части записей;
- вы можете объяснить, чем этот сценарий отличается от обычного одноразового скрипта.
