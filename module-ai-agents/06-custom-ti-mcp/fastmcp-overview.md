# FastMCP: что это такое

FastMCP позволяет писать MCP-серверы на Python без ручной работы с протоколом. Вы описываете обычные функции, а FastMCP делает их инструментами для агента.

## Что такое MCP

MCP, или Model Context Protocol, это способ подключать внешние инструменты и источники контекста к агенту через единый протокол.

## Что такое FastMCP

FastMCP — Python-библиотека для MCP-сервера. Вместо ручного описания протокола вы пишете функцию и помечаете ее строкой `@mcp.tool`.

Минимальный пример:

```python
from fastmcp import FastMCP

mcp = FastMCP("TI MCP Server")

@mcp.tool
def ping() -> str:
    """Health check tool."""
    return "pong"

if __name__ == "__main__":
    mcp.run()
```

## Почему это удобно

В этом примере важно не читать весь код построчно, а увидеть три элемента:

- функция `ping()` возвращает простой ответ;
- `@mcp.tool` делает функцию доступной агенту;
- строка `mcp.run()` запускает сервер.

FastMCP удобен тем, что:

- функции остаются обычными функциями;
- декоратор превращает их в инструменты MCP;
- type hints, или подсказки типов, описывают входные и выходные данные;
- docstring, или короткое описание функции, помогает агенту понять назначение инструмента.

Агенту нужен не только код, но и понятное описание того, когда вызывать инструмент и какой результат ждать.

## Что делает `@mcp.tool`

Декоратор `@mcp.tool` сообщает FastMCP, что обычная Python-функция должна стать инструментом MCP. После этого агент видит ее как вызываемую команду, а не как внутреннюю функцию вашего файла.

Идея простая: вы пишете функцию, FastMCP делает ее доступной агенту, OpenCode вызывает ее как инструмент.

## Как type hints превращаются в схему инструмента

Type hints — это подсказки типов в сигнатуре функции. Они показывают, что функция принимает и что возвращает.

Когда вы пишете:

```python
def lookup_ioc(ioc: str) -> dict:
```

FastMCP использует сигнатуру функции, чтобы понять:

- как называется аргумент;
- какой тип ожидается;
- какой формат результата возвращается.

Для агента это означает более понятную схему вызова, а для вас снижает шанс ошибиться в формате входных данных.

## Зачем важен docstring

Docstring — это текстовое описание внутри функции. В MCP-инструменте он работает как короткая инструкция для агента:

- когда использовать этот инструмент;
- какие данные передавать;
- что вернется на выходе.

Если docstring короткий и понятный, агент лучше понимает назначение инструмента. Если он пустой или слишком расплывчатый, поведение становится хуже.

## Что не нужно делать

Не пишите низкоуровневую MCP-обвязку вручную, если задача обычная.

Инструмент лучше держать узким:

- одна задача;
- понятный вход;
- понятный выход;
- короткое описание.
