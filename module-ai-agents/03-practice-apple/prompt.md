# Промпт для агента: Apple macOS CVE за 2026 год

Ниже рабочий промпт для агента. Его можно использовать без изменений или адаптировать под другой источник.

## Промпт

```text
Ты работаешь как исследователь Threat Intelligence и автоматизируешь сбор данных из официальных бюллетеней Apple.

Твоя задача: получить все CVE, связанные с macOS, опубликованные в 2026 году на официальной странице Apple Security Releases:
https://support.apple.com/en-us/100100

Требования:

1. Используй только официальный источник Apple.
2. Найди все записи за 2026 год.
3. Отфильтруй только те записи, которые относятся к macOS.
4. Из каждой подходящей записи извлеки:
   - название продукта;
   - дату публикации;
   - ссылку на запись;
   - все CVE, упомянутые в записи.
5. Если CVE перечислены в тексте, извлекай их регулярным выражением вида CVE-\d{4}-\d{4,7}.
6. Удали дубли CVE внутри одной записи и дубли записей между страницами.
7. Сохрани результат в JSON-файл.
8. Для каждой записи используй структуру:
   {
     "id": "uuid",
     "href": "...",
     "product": "...",
     "cves": []
   }
9. Если у записи нет CVE, все равно сохрани ее, но cves должен быть пустым массивом.
10. Не добавляй данные из других сайтов.
11. Не выдумывай CVE. Если CVE нет в тексте, оставь массив пустым.
12. Если страница разбита на несколько экранов или страниц, переходи по всем релевантным элементам интерфейса, пока не соберешь все записи за 2026 год.
13. После сбора данных проверь JSON на валидность.
14. В конце выведи краткую сводку:
    - сколько записей найдено;
    - сколько уникальных CVE извлечено;
    - путь к сохраненному JSON.

Работай аккуратно, по шагам, и не пропускай проверку результата.
```

## Практические команды

Проверка текста промпта:

```bash
cat module-ai-agents/03-practice-apple/prompt.md
```

Ожидаемый результат: терминал выводит промпт, который можно передать агенту.

Проверка сохранения JSON:

```bash
ls -la
```

Ожидаемый результат: в списке файлов виден JSON, который создал агент.
