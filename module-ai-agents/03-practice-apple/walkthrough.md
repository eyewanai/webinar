# Разбор практики: как агент собирает macOS CVE за 2026 год

Этот разбор нужен, чтобы студент видел не только финальный результат, но и логику выполнения. Для Threat Intelligence это особенно важно: если вы понимаете шаги агента, вы можете проверить его ошибки и адаптировать сценарий под другой источник.

Официальный источник:

- [Apple Security Releases](https://support.apple.com/en-us/100100)

## Что делает агент

Агент выполняет задачу как последовательность действий:

1. Открывает официальный список security releases.
2. Фильтрует записи за 2026 год.
3. Находит записи, относящиеся к macOS.
4. Переходит в каждую релевантную запись.
5. Извлекает CVE из текста.
6. Удаляет дубли.
7. Сохраняет данные в JSON.

## Как агент навигирует

Навигация обычно строится так:

- открыть главную страницу с бюллетенями;
- найти нужный год;
- определить записи, где в названии есть macOS;
- открыть каждую запись отдельно, если на сводной странице не хватает деталей;
- повторить до завершения обхода.

Если используется браузерная автоматизация, агент не «угадывает» содержимое, а работает по элементам страницы и доступному тексту.

## Как извлекаются CVE

Для извлечения CVE обычно используют регулярное выражение:

```regex
CVE-\d{4}-\d{4,7}
```

Почему именно так:

- `CVE-` фиксирует формат идентификатора;
- `\d{4}` — год;
- `\d{4,7}` — порядковый номер, который может иметь разную длину.

После извлечения агент:

- собирает все совпадения;
- удаляет повторения;
- приводит их к массиву строк;
- сохраняет в структуре записи.

## Как происходит обогащение

Обычно обогащение означает, что к каждой записи добавляются дополнительные поля:

- нормализованный идентификатор;
- ссылка на источник;
- продукт;
- дата публикации;
- сопоставление с внешними источниками;
- сведения о том, есть ли запись в других базах.

Для этой практики базовое обогащение можно ограничить официальным источником и валидацией структуры. Следующий шаг — подключить MITRE, NVD или CISA KEV.

## Как сохраняется JSON

Пример структуры:

```json
{
  "id": "uuid",
  "href": "...",
  "product": "...",
  "cves": []
}
```

### Что означает каждое поле

- `id` — уникальный идентификатор записи внутри вашего датасета. Обычно это UUID, чтобы каждый объект был однозначно адресуемым.
- `href` — ссылка на конкретную страницу бюллетеня, из которой извлечены данные.
- `product` — название продукта, например macOS Sequoia, macOS Sonoma или другой macOS-релиз.
- `cves` — массив CVE-идентификаторов, найденных в тексте записи.

## Практические команды

Проверка JSON после сохранения:

```bash
python -m json.tool result.json
```

Проверка наличия CVE-паттерна в тексте:

```bash
grep -Eo 'CVE-[0-9]{4}-[0-9]{4,7}' result.json
```

## Что должно получиться

- вы понимаете последовательность действий агента;
- вы можете объяснить логику навигации по странице;
- вы понимаете, как regex извлекает CVE;
- вы понимаете, как формируется итоговый JSON.

## Проверка результата

- проверьте, что JSON валиден;
- проверьте, что в `cves` лежат только строковые CVE-идентификаторы;
- убедитесь, что ссылки ведут на официальный источник;
- проверьте, что нет дублей внутри одной записи.
