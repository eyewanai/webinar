# Установка OpenCode на macOS

Этот раздел показывает, как установить OpenCode на macOS так, чтобы у студента был повторяемый и проверяемый сценарий. Для учебной группы это важно: одинаковая база уменьшает количество случайных ошибок на старте.

## Что понадобится

- macOS;
- установленный терминал;
- Node.js;
- npm;
- доступ в интернет;
- права на установку пользовательских пакетов.

Официальные ссылки:

- [OpenCode Intro](https://opencode.ai/docs/)
- [Node.js Downloads](https://nodejs.org/en/download)
- [npm Documentation](https://docs.npmjs.com/)
- [OpenCode Download](https://opencode.ai/download)

## Шаг 1. Установите Node.js

OpenCode можно ставить через npm, поэтому сначала нужен Node.js.

Проверьте, установлен ли Node.js:

```bash
node -v
```

Проверьте npm:

```bash
npm -v
```

Если команды не найдены, установите Node.js с официального сайта:

```bash
# Скачайте установщик с официальной страницы Node.js
```

## Шаг 2. Установите OpenCode через npm

Официальная команда установки:

```bash
npm install -g opencode-ai
```

Почему именно так:

- команда повторяет официальный путь из документации;
- установка через npm удобна для учебной среды;
- это минимально зависимый способ без дополнительных менеджеров пакетов.

## Шаг 3. Проверьте запуск

После установки проверьте, что бинарник доступен:

```bash
opencode --help
```

Если OpenCode установлен корректно, вы увидите справку по командам и параметрам.

## Шаг 4. Первый запуск

Запустите OpenCode в каталоге учебного проекта или отдельной рабочей папки:

```bash
opencode
```

Если OpenCode требует выбора модели или авторизации, это нормально: на этом этапе вы проверяете, что сам инструмент стартует.

## Шаг 5. Обновление

Если вы ставили OpenCode через npm, обновление обычно выполняется той же глобальной установкой:

```bash
npm install -g opencode-ai
```

Это перезапишет установленную версию на новую, если пакет обновился.

## Альтернатива через Homebrew

Если в вашей среде принято использовать Homebrew, официальная документация OpenCode также показывает установку через tap:

```bash
brew install anomalyco/tap/opencode
```

Это полезно, если вы уже используете Homebrew как стандартный пакетный менеджер в лаборатории.

## Практические команды

```bash
node -v
npm -v
npm install -g opencode-ai
opencode --help
opencode
```

## Что должно получиться

- `node` и `npm` доступны из терминала;
- `opencode` запускается без ошибки «command not found»;
- вы можете открыть справку OpenCode;
- у вас есть установленный агентный клиент для дальнейшей настройки.

## Проверка результата

- выполните `node -v` и `npm -v`;
- выполните `opencode --help`;
- убедитесь, что в выводе нет ошибок установки;
- при необходимости проверьте официальный раздел загрузок OpenCode.
