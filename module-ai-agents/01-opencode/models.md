# Модели в OpenCode

Качество агентного сценария зависит от модели: насколько она держит задачу, вызывает инструменты и не ломает последовательность действий.

## С чего начать

OpenCode поддерживает разные провайдеры и локальные модели, поэтому агент не привязан к одному API.

Ссылки:

- [OpenCode Models](https://opencode.ai/docs/models/)
- [OpenCode Providers](https://opencode.ai/docs/providers/)
- [OpenCode Download](https://opencode.ai/download)

## Какие модели можно использовать

В OpenCode можно использовать:

- облачные модели от популярных провайдеров;
- локальные модели, если они доступны в вашей среде;
- модели, подключенные через поддерживаемого провайдера;
- модели, доступные через конфигурацию OpenCode.

Запоминать список названий не нужно. Важно выбрать модель по критериям:

- качество рассуждения;
- качество вызова инструментов;
- скорость;
- цена;
- доступность в вашей организации;
- совместимость с вашей задачей.

## Какие модели бесплатные

Бесплатные варианты в агентном сценарии обычно бывают двух типов:

- локальная модель, если вы запускаете ее у себя;
- модель, доступная через провайдера с бесплатным планом или trial.

Важно различать:

- сам OpenCode может быть бесплатным как инструмент;
- использование облачной модели может стоить денег;
- локальная модель может не требовать API-ключа, но потребует вычислительных ресурсов.

Если смотреть на OpenCode Zen, официальная документация отдельно перечисляет бесплатные модели, доступные на момент публикации страницы. В документации встречаются, например:

- `Big Pickle`;
- `GLM 4.7 Free`;
- `Kimi K2.5 Free`;
- `MiniMax M2.1 Free`.

Важно: бесплатные варианты могут меняться. Проверяйте актуальный список в официальном Zen-документе.

## Какие модели требуют API-ключ

Если модель работает через облачного провайдера, почти всегда нужен API-ключ. В OpenCode ключи обычно подключаются через команду `/connect` или через переменные окружения в конфиге.

Пример из документации OpenCode:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "openai": {
      "options": {
        "apiKey": "{env:OPENAI_API_KEY}"
      }
    }
  }
}
```

## Ограничения бесплатных моделей

Бесплатная модель не всегда подходит для агентных задач. Типовые ограничения:

- ограничение по количеству запросов;
- ограничение по скорости;
- ограничения на вызов инструментов;
- меньшая стабильность при длинных цепочках действий;
- меньшее окно контекста;
- возможные ограничения по качеству ответа на сложные задачи.

В TI-задачах это особенно заметно, потому что задача часто включает:

- навигацию по большим страницам;
- извлечение шаблонов;
- повторяющиеся проверки;
- обогащение данных;
- сохранение результата.

Если модель плохо вызывает инструменты, пайплайн становится ненадежным.

## Где получить API-ключи

Получайте ключи только из официальных кабинетов провайдера.

Обычно всё выглядит так:

1. Зайдите в кабинет провайдера.
2. Создайте API key.
3. Сохраните его в переменной окружения.
4. Подключите его в конфиг OpenCode.

Примеры официальных стартовых страниц:

- [OpenAI API docs](https://platform.openai.com/docs)
- [Anthropic docs](https://docs.anthropic.com/)
- [Google AI documentation](https://ai.google.dev/)
- [OpenCode Zen](https://opencode.ai/docs/zen/)

## Как выбирать модель для практики

Для практики выбирайте модель, которая:

- хорошо работает с инструментами;
- не слишком дорогая для ваших тестов;
- доступна в вашей среде;
- умеет последовательно выполнять действия.

Для практики важнее надежное выполнение сценария, чем абстрактно «самая умная» модель.

## Практические команды

Посмотреть список доступных моделей:

```bash
opencode /models
```

Ожидаемый результат: OpenCode показывает модели, доступные для подключения.

Проверить выбранную модель в конфиге:

```bash
cat ~/.config/opencode/opencode.json
```

Ожидаемый результат: в JSON видно поле `model` или настройки провайдера.

Проверить переменную окружения:

```bash
echo "$OPENAI_API_KEY"
```

Ожидаемый результат: терминал выводит значение ключа или пустую строку, если ключ не задан.
