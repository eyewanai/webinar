# OpenCode: агент в терминале

OpenCode нужен, когда модель должна не только отвечать, но и выполнять действия в рабочей среде: читать конфиг, вызывать инструменты, запускать команды и сохранять файлы.

## Зачем нужен OpenCode

Обычный ChatGPT подходит для объяснений и анализа текста. OpenCode добавляет к модели агентную оболочку:

- доступ к инструментам;
- доступ к локальному контексту проекта;
- возможность подключать MCP-серверы.

В Threat Intelligence это полезно, когда нужно:

- открыть страницу бюллетеня;
- собрать уязвимости;
- отфильтровать лишнее;
- сохранить JSON;
- передать результат следующему этапу пайплайна.

## Чем OpenCode отличается от обычного ChatGPT

OpenCode отличается способом работы, а не «уровнем интеллекта».

- ChatGPT отвечает в чате.
- OpenCode работает как агент в терминале.
- ChatGPT без внешних инструментов ограничен текстом.
- OpenCode может вызывать инструменты, если вы их разрешили.
- ChatGPT не знает структуру вашего локального проекта.
- OpenCode читает конфигурацию и может работать в окружении пользователя.

Если задача заканчивается на «объясни», достаточно чата. Если нужно «собери, сохрани, обнови, проверь», удобнее агент.

## Зачем нужны инструменты

Инструмент — это внешняя функция или сервис, который агент может вызвать. Инструменты нужны для действий, которые модель сама не выполняет:

- читать и изменять файлы;
- выполнять команды;
- открывать веб-страницы;
- использовать браузер;
- обращаться к API и MCP-сервисам.

Без инструментов агент остается справкой. С инструментами он становится исполнителем.

## Зачем нужен MCP

MCP, или Model Context Protocol, задает единый способ подключать инструменты к агенту. Не нужно описывать каждую интеграцию заново: сервер сообщает агенту, какие действия доступны и как их вызвать.

В практической работе это означает:

- вы можете подключить Playwright MCP для браузерной автоматизации;
- вы можете подключить собственный локальный MCP для парсинга файлов;
- вы можете подключить удаленный MCP для доступа к сервисам и API.

## Практические команды

Проверка базовой установки OpenCode:

```bash
opencode --help
```

Ожидаемый результат: OpenCode выводит справку по командам и параметрам.

Проверка доступных моделей:

```bash
opencode /models
```

Ожидаемый результат: OpenCode открывает список моделей, доступных в вашей среде.

Проверка конфигурации:

```bash
cat ~/.config/opencode/opencode.json
```

Ожидаемый результат: терминал выводит содержимое конфига или сообщает, что файл еще не создан.

Проверка доступных MCP-серверов:

```bash
opencode mcp list
```

Ожидаемый результат: OpenCode показывает подключенные MCP-серверы или пустой список.

## Быстрые действия

### Как выбрать модель

Откройте список доступных моделей:

```bash
opencode /models
```

Ожидаемый результат: вы видите список моделей и можете выбрать одну для практики.

Дальше выберите модель, которая подходит для задачи и доступна в вашей среде. Для учебных заданий обычно достаточно одной рабочей модели, а не целого набора.

### Как включить или выключить MCP-сервер

Откройте `opencode.json` и поменяйте у нужного сервера поле `enabled`:

```json
{
  "mcp": {
    "playwright": {
      "enabled": true
    },
    "ti-tools": {
      "enabled": false
    }
  }
}
```

- `true` включает сервер;
- `false` выключает сервер.

После изменения конфига перезапустите OpenCode.

### Что обычно нужно для выполнения задания

Чаще всего хватает только трех вещей:

1. выбрать модель;
2. включить нужные MCP-серверы;
3. запустить OpenCode заново.
