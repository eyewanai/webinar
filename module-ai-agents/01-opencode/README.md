# OpenCode: агент в терминале

OpenCode нужен тогда, когда вы хотите не просто получить ответ от модели, а поручить ей последовательность действий с инструментами: читать конфиг, обращаться к MCP-серверам, запускать команды, работать с файлами и поддерживать контекст между шагами.

## Зачем нужен OpenCode

Обычный ChatGPT полезен для объяснений, генерации текста и анализа, но у него нет прямого доступа к вашей среде выполнения. OpenCode добавляет агентную оболочку поверх модели и дает три важных свойства:

- доступ к инструментам;
- доступ к локальному контексту проекта;
- возможность подключать MCP-серверы.

Именно это делает его полезным в Threat Intelligence. Агент может не только объяснить, что такое CVE, но и:

- открыть страницу бюллетеня;
- собрать уязвимости;
- отфильтровать лишнее;
- сохранить JSON;
- передать результат следующему этапу пайплайна.

## Чем OpenCode отличается от обычного ChatGPT

OpenCode отличается не «умнее/глупее», а способом работы.

- ChatGPT отвечает в чате.
- OpenCode работает как агент в терминале.
- ChatGPT без внешних инструментов ограничен текстом.
- OpenCode может вызывать инструменты, если вы их разрешили.
- ChatGPT не знает структуру вашего локального проекта.
- OpenCode читает конфигурацию и может работать в окружении пользователя.

Практический вывод простой: если задача заканчивается на «объясни», достаточно чата. Если задача заканчивается на «собери, сохрани, обнови, проверь, обогати», нужен агент.

## Зачем нужны инструменты

Инструменты нужны для выполнения действий, которые модель сама сделать не может:

- читать и изменять файлы;
- выполнять команды;
- открывать веб-страницы;
- использовать браузер;
- обращаться к API и MCP-сервисам.

Без инструментов агент легко превращается в «говорящую справку». С инструментами он становится исполнителем.

## Зачем нужен MCP

MCP позволяет подключать внешние инструменты по стандартному протоколу. Это важно, потому что не нужно писать отдельную интеграцию под каждый новый сервис. Вы описываете сервер один раз и подключаете его в агенте.

В практической работе это означает:

- вы можете подключить Playwright MCP для браузерной автоматизации;
- вы можете подключить собственный локальный MCP для парсинга файлов;
- вы можете подключить удаленный MCP для доступа к сервисам и API.

## Практические команды

Проверка базовой установки OpenCode:

```bash
opencode --help
```

Проверка доступных моделей:

```bash
opencode /models
```

Проверка конфигурации:

```bash
cat ~/.config/opencode/opencode.json
```

Проверка доступных MCP-серверов:

```bash
opencode mcp list
```

## Что должно получиться

- вы понимаете, когда нужен чат, а когда нужен агент;
- вы понимаете, зачем агенту инструменты;
- вы понимаете, почему MCP — это не «еще одна библиотека», а слой интеграции;
- у вас есть базовая карта разделов модуля.

## Проверка результата

- вы можете своими словами объяснить разницу между OpenCode и обычным чат-режимом;
- вы можете перечислить минимум три задачи, где агент полезнее обычного чата;
- вы можете открыть официальную документацию OpenCode и MCP.
