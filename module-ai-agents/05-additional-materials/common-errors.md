# Частые ошибки студентов

Этот файл нужен как рабочая памятка для преподавателя и студента. В агентных задачах ошибки повторяются почти всегда, поэтому их полезно описывать заранее.

## 1. Неправильный JSON

### Как выглядит проблема

- пропущены кавычки;
- лишние запятые;
- перепутаны типы значений;
- JSON не проходит валидацию.

### Как проверять

```bash
python -m json.tool result.json
```

## 2. MCP не запускается

### Как выглядит проблема

- сервер не виден в OpenCode;
- клиент не находит конфиг;
- команда запуска неверная;
- сервер падает при старте.

### Как проверять

```bash
cat ~/.config/opencode/opencode.json
opencode --help
```

## 3. Браузер не установлен

### Как выглядит проблема

- Playwright MCP не может открыть браузер;
- `browser not installed`;
- сервер просит установить Playwright browser binaries.

### Как проверять

```bash
npx -y @playwright/mcp@latest --browser=chromium --headless
```

## 4. Агент не может найти элемент

### Как выглядит проблема

- страница изменила разметку;
- элемент скрыт;
- селектор слишком хрупкий;
- агент ожидает другой текст.

### Как проверять

- открыть страницу вручную;
- сверить текст;
- убедиться, что агент ориентируется на актуальную страницу.

## 5. CVE дублируются

### Как выглядит проблема

- одна и та же CVE встречается несколько раз;
- одна запись попала в итог дважды;
- дубль появился при обходе страниц.

### Как проверять

```bash
grep -Eo 'CVE-[0-9]{4}-[0-9]{4,7}' result.json | sort -u
```

## 6. Источник неофициальный

### Как выглядит проблема

- данные взяты из вторичного блога;
- пропущена ссылка на первоисточник;
- у студента нет доверенной базы для проверки.

### Как проверять

- открыть официальный сайт производителя;
- сверить URL;
- убедиться, что источник указан прямо в задании.

## Практические команды

```bash
python -m json.tool result.json
cat ~/.config/opencode/opencode.json
npx -y @playwright/mcp@latest --browser=chromium --headless
```
