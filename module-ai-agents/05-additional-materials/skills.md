# Skills: переиспользуемые инструкции для агента

Skills нужны, когда один и тот же сценарий не хочется заново описывать в каждом промпте.

Defanged IOC — безопасная запись индикатора, например `example[.]com`. Enrichment — обогащение IOC данными из внешнего источника.

Это удобно, когда задача повторяется:

- одинаковый стиль проверки;
- одинаковая последовательность действий;
- одинаковые правила извлечения и нормализации;
- одинаковые ограничения по источникам и форматам.

Вместо длинного промпта агент получает короткое указание, какой skill использовать.

## Что такое skill

Skill — это упакованная инструкция для агента.

Обычно в skill лежит файл `SKILL.md`. Внутри описывают:

- что делает skill;
- когда его использовать;
- какие шаги он выполняет;
- какие ограничения у него есть;
- какие данные ему нужны на входе.

Идея: повторяющиеся правила хранятся в одном файле, а не копируются в каждый промпт.

## Зачем это нужно

Skills полезны, когда один и тот же сценарий запускается много раз.

Например:

- извлечение IOC из статей;
- нормализация defanged IOC;
- обогащение через API;
- проверка JSON;
- одинаковая структура ответа для разных источников.

Если такой сценарий оформить как skill, агент будет подхватывать его как готовую рабочую инструкцию.

## Как это устроено в целом

OpenCode находит skills по имени и загружает их тогда, когда они действительно нужны.

На практике это означает:

1. Вы создаете skill в проекте или в домашнем каталоге.
2. Описываете его в `SKILL.md`.
3. Разрешаете агенту пользоваться skill, если это нужно.
4. Вызываете skill из OpenCode, когда задача подходит под его назначение.

Такой подход сокращает промпты и делает поведение агента предсказуемее.

## OpenCode

В OpenCode skills ищутся в специальных папках.

### Где лежат файлы

Для проекта:

```text
.opencode/skills/<name>/SKILL.md
```

Для домашнего каталога:

```text
~/.config/opencode/skills/<name>/SKILL.md
```

OpenCode также умеет читать совместимые каталоги `.claude/skills/` и `.agents/skills/`, если они есть в проекте или в домашнем каталоге.

### Как выглядит skill

У каждого skill должен быть `SKILL.md` с YAML frontmatter. Минимально нужны:

- `name`;
- `description`.

Пример структуры:

```markdown
---
name: ti-ioc-enrichment
description: Extract, normalize, and enrich IOC from threat intelligence articles
compatibility: opencode
---

## Что делает skill

- находит IOC в тексте;
- нормализует defanged значения;
- передает список на enrichment;
- сохраняет результат в JSON.
```

### Как включить skill в OpenCode

Самый простой путь:

1. Создать `.opencode/skills/<name>/SKILL.md`.
2. Перезапустить OpenCode.
3. Проверить, что skill виден агенту.

Если в `opencode.json` у вас явно отключен инструмент `skill`, верните его обратно в рабочий режим. Обычно для этого достаточно убрать `skill: false` из настроек агента или заменить его на разрешающую конфигурацию.

Если skill должен быть доступен агенту, но вы хотите контролировать доступ, используйте permissions в `opencode.json`.

Пример для разрешения skill:

```json
{
  "permission": {
    "skill": {
      "*": "allow"
    }
  }
}
```

Если нужен более строгий режим, вместо `allow` можно использовать `ask` или `deny` для отдельных шаблонов.

### Как отключить skill

Если skill не должен использоваться в конкретной конфигурации, его можно скрыть через настройки агента или через permissions.

Пример отключения инструмента skill для встроенного агента:

```json
{
  "agent": {
    "plan": {
      "tools": {
        "skill": false
      }
    }
  }
}
```

Это полезно, если вы хотите, чтобы агент работал только с MCP-инструментами и не подхватывал дополнительные инструкции.

### Когда skill лучше, чем обычный промпт

Skill удобнее, если сценарий повторяется.

Обычный промпт хорош для разовой задачи. Skill лучше, если вы:

- регулярно извлекаете IOC из похожих источников;
- хотите единые правила обработки;
- не хотите каждый раз переписывать одну и ту же последовательность действий.

## Практические команды

Проверить, что skill-файл лежит в нужной папке:

```bash
find .opencode -maxdepth 3 -name "SKILL.md"
```

Ожидаемый результат: команда выводит путь к `SKILL.md`, если skill создан.

Проверить конфиг OpenCode:

```bash
cat ~/.config/opencode/opencode.json
```

Ожидаемый результат: терминал выводит конфиг, где можно проверить permissions и настройки агента.

Проверить, что skill можно включать через permissions:

```bash
grep -n '"skill"' ~/.config/opencode/opencode.json
```

Ожидаемый результат: команда показывает строки с настройками `skill` или ничего не выводит, если таких настроек нет.
