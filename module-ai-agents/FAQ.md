# FAQ и типовые ошибки

Короткий справочник для диагностики: сервер не запускается, агент выбирает не тот инструмент или результат выглядит подозрительно.

## Что здесь искать

Enrichment, или обогащение, здесь означает запрос к внешнему источнику за дополнительными сведениями об IOC.

FastMCP — Python-библиотека, на которой в курсе построен собственный MCP-сервер.

- как проверить, что MCP-сервер подключен;
- почему агент не вызывает нужный инструмент;
- как понять, что проблема в конфиге, а не в данных;
- что делать, если IOC не извлекаются;
- как быстро проверить результат после enrichment.

## Почему в курсе нужны и Playwright MCP, и TI MCP

Playwright MCP нужен для браузера. Он открывает страницу, получает текст и дает агенту доступ к содержимому сайта.

TI MCP нужен для задач Threat Intelligence: извлечь IOC, нормализовать их, обогатить через API и сохранить результат.

Если в практике участвует только один сервер, пайплайн распадается. Агент либо не получает текст страницы, либо не может обработать IOC дальше.

## Почему MCP-сервер лучше, чем один Python-скрипт

Python-скрипт решает одну задачу последовательно.

MCP-сервер превращает те же функции в инструменты, которые агент может вызывать по ходу работы:

- сначала открыть источник;
- потом извлечь данные;
- потом обогатить их;
- потом сохранить результат.

Это удобнее, когда источник меняется, а сам сценарий остается похожим.

## Что делать, если OpenCode не видит `ti-tools`

Проверьте три вещи:

- сервер добавлен в `opencode.json`;
- путь к `server.py` абсолютный;
- Python-окружение активируется без ошибок.

Если `ti-tools` используется как remote MCP, сервер должен быть запущен и отвечать на `http://127.0.0.1:8000/mcp`.

## Что делать, если `playwright` есть, а `ti-tools` нет

Это обычно означает, что включен только один сервер.

Проверьте конфиг OpenCode и убедитесь, что в нем одновременно есть:

- `playwright`;
- `ti-tools`.

Для практики по TI этого действительно достаточно. Другие серверы не нужны.

## Что делать, если сервер FastMCP запускается, но OpenCode его не подхватывает

Сначала проверьте, что сервер вообще отвечает локально.

```bash
curl -i http://127.0.0.1:8000/mcp
```

Ожидаемый результат: сервер возвращает HTTP-ответ. Если ответа нет, проблема в самом сервере или порте.

Если ответа нет, проблема в самом сервере. Если ответ есть, но OpenCode не видит сервер, обычно ошибка в конфиге или в URL.

Если конфиг меняли уже после запуска OpenCode, перезапустите клиент. Изменения не всегда подхватываются на лету.

## Что делать, если OpenCode видит сервер, но агент не использует его

Проверьте, что включены оба сервера:

- `playwright`;
- `ti-tools`.

Для этой практики они работают вместе. Один открывает статью, второй обрабатывает IOC.

## Нужно ли запускать `ti-tools` вручную каждый раз

Если `ti-tools` подключен как remote MCP на `localhost`, сервер нужно запускать отдельно.

Последовательность простая:

1. поднять FastMCP-сервер;
2. запустить OpenCode;
3. дать агенту задачу;
4. проверить результат.

## Почему после правки `opencode.json` ничего не меняется

Чаще всего OpenCode уже был запущен до изменения конфига.

Проверьте порядок:

1. сохранить файл;
2. закрыть OpenCode;
3. открыть его заново;
4. повторно проверить список MCP-серверов;
5. снова запустить практику.

## Что делать, если `RST_THREAT_API_KEY is not set`

Это значит, что переменная окружения не установлена в текущей сессии.

```bash
export RST_THREAT_API_KEY="your-api-key-here"
echo "$RST_THREAT_API_KEY"
```

Ожидаемый результат: вторая команда выводит значение ключа.

Если значение не выводится, API-ключ не попадет в запросы enrichment.

## Почему нельзя хранить API-ключ в коде

Потому что код часто:

- попадает в git;
- отправляется коллегам;
- копируется между машинами;
- логируется и индексируется внешними инструментами.

Переменная окружения проще для локальной разработки и безопаснее для учебного проекта.

## Почему IOC могут не находиться

Чаще всего причина одна из трех:

- в источнике нет IOC в нужном формате;
- регулярное выражение слишком узкое;
- текст страницы получен не полностью.

Если извлечение идет по defanged-формату, убедитесь, что в тексте действительно есть записи вида `example[.]com` или `1[.]2[.]3[.]4`.

## Что делать, если enrichment возвращает пустые поля

Проверьте:

- нормализуется ли IOC перед запросом;
- доступен ли API;
- не уперлись ли вы в лимиты;
- не вернул ли сервис ответ без нужных полей.

Пустой `description` или отсутствующий `src.report` не должны ломать весь пайплайн. В хорошей реализации это обрабатывается как обычный неполный ответ.

## Почему в JSON появляются дубликаты

Обычно это происходит, если один и тот же IOC встречается:

- на одной странице несколько раз;
- в разных частях статьи;
- в списке и в тексте одновременно.

Перед сохранением результат лучше дедуплицировать и уже потом писать в файл.

## Что делать, если JSON не проходит проверку

Проверьте файл стандартной командой:

```bash
python3 -m json.tool ioc.json
```

Ожидаемый результат: команда выводит отформатированный JSON или ошибку синтаксиса.

Если команда падает, чаще всего проблема в:

- лишней запятой;
- неэкранированных символах;
- оборванной структуре;
- неверной кодировке.

## Почему агент может выбрать не тот инструмент

Обычно это связано не с кодом, а с описанием инструмента.

Docstring — это короткое описание функции в коде, которое агент использует как подсказку.

Если docstring слишком общий, агенту сложнее понять:

- когда именно вызывать инструмент;
- какие данные туда передавать;
- что он получит на выходе.

В таких случаях помогает более короткое и точное описание.

## Практические команды

Проверить ключ:

```bash
echo "$RST_THREAT_API_KEY"
```

Ожидаемый результат: терминал выводит ключ или пустую строку, если он не задан.

Проверить конфиг OpenCode:

```bash
cat ~/.config/opencode/opencode.json
```

Ожидаемый результат: терминал выводит конфиг, где можно проверить секцию `mcp`.

Проверить, что JSON валиден:

```bash
python3 -m json.tool ioc.json
```

Ожидаемый результат: команда выводит отформатированный JSON или ошибку синтаксиса.

Проверить локальный MCP endpoint:

```bash
curl -i http://127.0.0.1:8000/mcp
```

Ожидаемый результат: локальный сервер возвращает HTTP-ответ.
